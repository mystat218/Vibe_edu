# Product Requirements Document (PRD): 부산 축제 정보 웹사이트 (Busan Festival Hub)

본 문서는 부산광역시 축제 OpenAPI 정보를 활용하여 사용자 친화적이고 미려한 디자인을 갖춘 부산 축제 정보 안내 웹사이트(Next.js & Tailwind CSS 기반)를 개발하기 위한 제품 요구사항 정의서(PRD)입니다.

---

## 1. 프로젝트 개요 (Overview)

### 1.1 배경 및 목적
부산은 사계절 내내 다양한 축제(부산불꽃축제, 바다축제, 모래축제, 맥주축제 등)가 개최되는 대한민국 대표 관광 도시입니다. 하지만 흩어져 있는 축제 정보를 한눈에 파악하고, 위치, 시기, 테마별로 통합 조회하기 어려웠습니다.
본 프로젝트는 공공데이터포털의 **부산광역시 축제 정보 서비스 API**를 활용하여, 부산을 찾는 관광객과 시민들이 축제 일정을 편리하게 확인하고, 테마 및 지역구별 필터링, 지도 기반 위치 탐색, 관심 축제 저장 등의 기능을 제공하는 프리미엄 웹 서비스를 제공하고자 합니다.

### 1.2 핵심 가치 (Core Values)
1. **시각적 우수성 (Visual Excellence):** 단순한 정보 나열을 넘어, 활기찬 축제 분위기를 느낄 수 있는 세련된 HSL 컬러 테마, 부드러운 애니메이션, 반응형 레이아웃을 통해 프리미엄 사용자 경험(UX)을 선사합니다.
2. **정보의 유용성 (Usability):** 직관적인 구/군별 필터, 월별 필터, 실시간 검색, 대중교통 정보 및 장애인 편의시설 등 상세한 정보를 빠르게 탐색할 수 있습니다.
3. **편의성 (Convenience):** 모바일 퍼스트 대응 및 지도 기반 위치 시각화, 북마크 기능을 통해 실시간 현장 활용성을 극대화합니다.

## 2. 기술 스택 (Technology Stack)

* **프레임워크:** Next.js (App Router 기반)
* **언어:** TypeScript
* **스타일링:** Tailwind CSS & Vanilla CSS (고급 스타일링 및 모던 디자인 적용)
* **애니메이션:** Framer Motion (부드러운 마이크로 인터랙션 구현)
* **지도 라이브러리:** Leaflet.js 또는 Kakao/Naver Map (API 키 제약이 없고 SSR 친화적인 오픈소스 Leaflet 권장)
* **아이콘:** Lucide React
* **배포 플랫폼:** Vercel

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 메인 대시보드 (Main Dashboard)
* **히어로 섹션 (Hero Banner):** 
  * 부산의 역동적인 이미지를 담은 고해상도 비주얼 배포.
  * 검색 바 및 핵심 통계(진행 중인 축제 수 등) 노출.
* **통합 검색 및 필터 (Search & Filters):**
  * **텍스트 검색:** 축제 이름, 태그, 내용 실시간 검색.
  * **구/군 필터:** 해운대구, 수영구, 영도구 등 부산시 16개 구/군별 다중 선택/필터링.
  * **월별 필터:** 1월 ~ 12월 개최 월별 빠른 필터링.
  * **요금 필터:** 무료 / 유료 구분 필터링.
* **축제 카드 리스트 (Festival Grid/List):**
  * 카드 형태로 썸네일 이미지, 축제 명(태그 정제), 개최 기간, 장소, 구/군 태그 제공.
  * 마우스 오버 시 부드러운 줌인 효과 및 상세 정보 프리뷰.
  * 로딩 상태 스켈레톤(Skeleton UI) 적용.

### 3.2 상세 정보 뷰 (Festival Detail View)
* **모달 또는 전용 상세 페이지:**
  * 모달 형태(대시보드에서 빠른 탐색 시)와 개별 공유 가능한 다이내믹 라우팅 상세 페이지(`[id]`) 동시 지원.
* **상세 정보 제공 항목:**
  * 축제 공식 명칭 (번역용 접미사 `(한,영, 중간,중번,일)` 제거 및 정제)
  * 캐치프레이즈 및 서브타이틀
  * 축제 소개 상세 내용 (`ITEMCNTNTS` 파싱하여 단락 구분 가독성 개선)
  * 기간 및 점등 시간 (`USAGE_DAY_WEEK_AND_TIME`, `USAGE_DAY` 결합 가독화)
  * 장소 및 주소, 상세 약도 정보 (`MAIN_PLACE`, `ADDR1`, `ADDR2`)
  * 대중교통 이용 안내 (`TRFC_INFO` 줄바꿈 및 가독성 개선)
  * 입장료/요금 (`USAGE_AMOUNT`가 비어있을 경우 '무료' 또는 '정보 없음' 자동 처리)
  * 문의 연락처 (`CNTCT_TEL`) 및 공식 홈페이지 바로가기 링크 (`HOMEPAGE_URL`)
  * **장애인 편의 정보:** `MIDDLE_SIZE_RM1` 데이터를 추출하여 배리어 프리(Barrier-free) 아이콘 및 팁으로 시각화.

### 3.3 지도 서비스 (Festival Map View)
* **인터랙티브 맵:**
  * 부산시 전체 지도를 띄우고 API의 `LAT`, `LNG` 좌표를 활용하여 축제 위치에 커스텀 마커(Marker) 표시.
  * 마커 클릭 시 축제 미니 카드(썸네일, 타이틀, 장소) 팝업 및 상세 페이지 이동 지원.
  * 구/군 필터 선택 시 지도 상 마커 실시간 필터링 적용.

### 3.4 관심 축제 북마크 (My Favorites)
* **북마크 기능:**
  * 각 카드 및 상세 페이지에 하트(Heart) 버튼 제공.
  * 로그인 없이 브라우저 로컬 저장소(`localStorage`)를 활용하여 가볍게 북마크 상태 유지.
  * 메인 대시보드 내 "내 북마크 모아보기" 토글 또는 별도 탭 제공.

## 4. 디자인 및 사용자 경험 (UX/UI Spec)

### 4.1 테마 및 디자인 톤앤매너
* **컨셉:** "Ocean Vista & Festive Energy"
* **색상 팔레트:**
  * **Primary:** 부산 바다를 상징하는 딥 오션 블루 (Deep Ocean Blue, `#0052CC`) 및 활기찬 에너지를 상징하는 네온 코랄 (Neon Coral, `#FF4B5C`)
  * **Background:** 세련된 다크 모드(Slate/Zinc 기반 다크 블루 톤) 및 모던한 라이트 모드 지원.
  * **Gradients:** 헤드라인과 주요 컴포넌트에 은은한 그라디언트 적용.
* **글꼴:** Google Fonts의 'Inter' 또는 'Outfit' (영문), 'Pretendard' 또는 'Noto Sans KR' (국문) 적용으로 가독성 극대화.
* **디자인 요소:** 글래스모피즘(Glassmorphism) 카드 스타일, 부드러운 외곽선 그림자, 아이콘 활용을 통한 시각적 가이드라인 구축.

### 4.2 마이크로 인터랙션
* 버튼 호버 시 스케일 업 및 그라디언트 이동 애니메이션.
* 상세 페이지 오픈 시 부드러운 페이드인 슬라이드 업 효과.
* 북마크 하트 클릭 시 튕기는 애니메이션(Spring Effect) 구현.

## 5. 데이터 인터페이스 (API Integration)

* **제공 API:** 부산광역시 축제 서비스 (`getFestivalKr`)
* **Endpoint URL:** `https://apis.data.go.kr/6260000/FestivalService/getFestivalKr`
* **요청 파라미터:**
  * `serviceKey`: `126fc93eb5c942b022866f746537cfa6b5c2f2cd141dad1d41e0fc14d0465e88`
  * `pageNo`: `1`
  * `numOfRows`: `100` (전체 축제를 한 번에 불러와 클라이언트 사이드에서 캐싱 및 필터링하여 매끄러운 반응성 보장)
  * `resultType`: `json`
* **응답 정제 규칙:**
  * **제목 정제:** `MAIN_TITLE` 필드에 포함된 `(한,영, 중간,중번,일)` 등의 다국어 괄호 표기 제거. (예: `부산바다축제(한,영, 중간,중번,일)` -> `부산바다축제`)
  * **이미지 Fallback:** `MAIN_IMG_NORMAL`이 누락되거나 로드되지 않을 경우, 부산 대표 이미지 또는 테마별 디폴트 일러스트레이션으로 대체.
  * **좌표 보정:** 일부 API 항목의 좌표가 유효하지 않을 경우를 대비하여 방어 코드 및 디폴트 부산 중심 좌표 설정.

## 6. 비기능적 요구사항 (Non-Functional Requirements)

1. **SEO 및 메타 태그 최적화:**
   * Next.js Metadata API를 사용하여 페이지별 적절한 타이틀과 오픈그래프(OG) 태그 적용.
   * "부산 축제 정보", "부산 여행 가이드" 등 주요 키워드를 메타 정보에 삽입하여 검색 엔진 노출 유도.
2. **성능 최적화:**
   * API 응답 결과는 정적/증분 정적 재생성(ISR) 또는 Next.js fetch 캐싱을 적용하여 페이지 로드 속도 1초 이하 유지.
   * 이미지 최적화(`next/image`)를 적극 활용하여 모바일 대역폭 절약.
3. **웹 접근성 (Accessibility):**
   * 시각장애인 스크린 리더용 `aria-label` 및 대체 텍스트(`alt`) 충실히 적용.
   * 명확한 대비를 통한 가독성 보장.
4. **반응형 디자인 (Responsive Layout):**
   * 모바일(Mobile), 태블릿(Tablet), 데스크톱(Desktop) 뷰포트에 완벽히 대응하는 반응형 그리드 설계.

## 7. 검증 및 테스트 계획

1. **로컬 기능 테스트:** 
   * 검색, 필터링(구/군, 월별) 기능이 클라이언트 단에서 오동작하지 않는지 확인.
   * 북마크의 저장 및 삭제, 새로고침 시 데이터 유지 여부 검증.
2. **배포 테스트:**
   * Vercel 배포 시 빌드 오류 및 환경변수 누락 여부 확인.
   * 모바일 실기기 브라우저(Safari, Chrome)에서의 레이아웃 및 맵 스크롤 적합성 검증.
